# 개발 일지

## Today 진행 내용 (약 2시간)
- **병원 가용 상태 UI 구현**
  - 응급실 일반 병상 가용 현황을 원형 프로그레스 UI로 시각화
  - 숫자 중심 정보에서 벗어나 직관적으로 상태를 인지할 수 있도록 개선
  - 거리순 / 정확도순 리스트에서 동일한 UI 컴포넌트 재사용

- **병원 리스트 및 추천 영역 코드 구조 개선**
  - 병원 데이터 요청 로직을 상위 컴포넌트로 통합
  - 위치 정보, 반경 설정을 Pinia 스토어 기준으로 단일 관리
  - 위치 변경 시에만 서버 요청 발생하도록 구조 정리
  - 반경 변경은 프론트엔드 필터링 방식으로 분리하여 UX 개선

- **병원 즐겨찾기 기능 구현**
  - 병원 리스트 카드에 즐겨찾기 버튼 UI 추가
  - 즐겨찾기 추가/해제 시 Optimistic UI 방식 적용
  - 사용자가 찜한 병원을 별도 페이지에서 확인 가능하도록 구성

- **병원 상세 정보 표시 기능 추가**
  - 병원 카드 내 “자세히 보기” 토글 기능 구현
  - 전화번호, 주소 등 추가 정보를 필요 시에만 노출
  - 네이버 지도로 바로 이동할 수 있는 외부 링크 제공

## 학습한 내용
- Pinia `persist` 사용 시 초기 상태 복원과 UI 렌더 타이밍 간 불일치가 발생할 수 있음을 경험
- 반응형 상태(`computed`, `watch`)와 초기 마운트 시점 동기화의 중요성 이해
- 서버 요청 로직과 프론트엔드 필터링 로직을 분리하는 구조의 장점 학습
  - 네트워크 요청 감소
  - 반경 변경 시 즉각적인 UI 반응 가능
- 공통 UI 컴포넌트를 활용한 상태 시각화 설계 경험

## 실패한 것 / 부족했던 점
- 반경 필터가 최초 렌더 시 정상 적용되지 않는 문제로 디버깅 시간이 소요됨
- Pinia 상태 초기화와 컴포넌트 마운트 시점에 대한 고려가 부족했음
- 기능 구현에 집중하면서 초기 데이터 흐름을 충분히 정리하지 못한 점이 아쉬움
